# Fairy Keys — Design

**Date:** 2026-07-03
**What it is:** A browser-based keyboard game for a 3 (almost 4) year old who knows some letters. Fairy/princess and colors/shapes theme. Teaches letter recognition, finding letters on the keyboard, and the keys real games use (arrows, space bar).

## Goals

- Fun first: every game is playful, with no way to lose and no negative feedback.
- Grows with her: games range from "press anything" to "find this letter" to steering with arrow keys.
- Simple to build and run: plain HTML/CSS/JavaScript, no frameworks, no build step. Opens in any browser.
- Published free on GitHub Pages so she can play on any computer with a keyboard.

## Structure

One folder of web files in its own GitHub repository:

- `index.html` — the picture menu: four big colorful buttons (no reading required), one per game.
- One page (or screen) per mini-game.
- Shared helpers for: speaking letters aloud (the browser's built-in speech voice), sparkle/firework effects, sound chimes (browser-generated), the star counter, the speed setting, and the fullscreen button.

## The four mini-games

1. **Magic Keys** — press any key: a sparkle burst in a random color and position. Letter and number keys are spoken aloud ("B!"); other keys make a magic chime. No goal, no failure. Built first.
2. **Find the Letter** — a fairy shows a big uppercase letter and the voice asks "Can you find B?" Correct key: fireworks and spoken praise. Any other key: a soft twinkle, never a buzzer. After a few misses, the letter gently wiggles on a small on-screen keyboard as a hint. Letters chosen at random from A–Z.
3. **Bubble Pop** — 2–3 bubbles with uppercase letters drift slowly up the screen. Pressing a matching letter pops that bubble in a sparkle shower. Missed bubbles float away off the top — no game over.
4. **Fairy Flight** — arrow keys steer a fairy around the screen to collect stars; space bar waves her wand, leaving a sparkle trail. No enemies, no way to lose. Teaches gaming keys.

## Look and sound

- Bright pastel fairy-tale palette; sparkles, bubbles, and characters drawn with simple shapes and emoji-style art (nothing to download or license).
- The browser's built-in speech voice speaks letter names and short praise ("Great job!").
- Gentle chimes and pops generated by the browser (Web Audio) — no sound files.
- A star counter ticks up during play, purely for delight.

## Settings

- **Speed slider** — turtle 🐢 to rabbit 🐇, controls how fast things move (bubble rise speed, fairy speed, general animation pace). Lives on the menu page; the browser remembers the choice between visits (localStorage). Start slow by default.
- No accounts, no profiles, no progress tracking.

## Toddler-proofing

- Every game works with single key presses only — never key combinations.
- Leaving a game requires a deliberate action: the Esc key or a small star button in a corner.
- A fullscreen toggle button (⛶) sits in the bottom-right corner of every screen, so stray clicks can't wander off the page.
- Known limitation, accepted: a web page cannot block the Windows key; worst case the Start menu opens.

## Build order and testing

1. Menu page + Magic Keys, with the shared sparkle/speech/sound helpers.
2. Publish to GitHub Pages as soon as Magic Keys works, so play can start early.
3. Find the Letter, then Bubble Pop, then Fairy Flight — each added one at a time.
4. Each game is verified in a browser preview during development (keys pressed, sparkles shown, speech triggered, speed slider respected). The final test is the granddaughter at the real keyboard.

## Out of scope (for now)

- Typing whole words or her name (possible later addition).
- Progress tracking, scores that persist, multiple player profiles.
- Mobile/tablet support — this is a keyboard game for a Windows PC.
